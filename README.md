# Go Task Runner 

Высокопроизводительный распределённый планировщик задач, реализованный на языке Go. Проект создан в образовательных целях для глубокого изучения устройства рантайма Go, конкурентного программирования и продвинутых архитектурных паттернов.

---

## 🚀 Ключевые особенности и изученные темы

### 🔹 Concurrency & Parallelism

В проекте реализован паттерн **Worker Pool**, позволяющий эффективно распределять задачи между несколькими воркерами (горутинами). Это обеспечивает:

* Параллельную обработку задач
* Контроль уровня конкурентности
* Повышение общей пропускной способности системы

---

### 🔹 Communication

Организовано взаимодействие между компонентами через:

* **Channels (каналы)** — как буферизированные, так и небуферизированные
* Использование оператора `select` для обработки нескольких каналов одновременно

---

### 🔹 Interface-driven Design

Архитектура построена на основе интерфейсов, что даёт:

* Возможность легко добавлять новые типы задач
* Независимость компонентов
* Простоту масштабирования и тестирования

---

### 🔹 Error Handling & Resilience

Для повышения устойчивости системы реализованы следующие механизмы:

* **Panic Recovery** внутри воркеров
  Позволяет избежать падения всего приложения при ошибке в отдельной задаче

* **Потокобезопасные метрики**
  Используется пакет `atomic` для безопасного обновления счётчиков

---

### 🔹 Memory Management

Оптимизация работы с памятью достигается за счёт:

* Использования `sync.Pool` для переиспользования объектов
* Снижения количества аллокаций
* Уменьшения нагрузки на Garbage Collector (GC)

---

### 🔹 Context & Lifecycle

Реализовано управление жизненным циклом задач:

* Использование `context.Context` для:

  * Таймаутов
  * Отмены задач

* **Graceful Shutdown (мягкое завершение)**:

  * Обработка системных сигналов (`os/signal`)
  * Корректное завершение всех воркеров

---

### 🔹 Profiling & Observability

Добавлены инструменты анализа производительности:

* Интеграция `pprof`:

  * CPU profiling
  * Heap profiling

* Визуализация:

  * Графы вызовов
  * Flame-графики

---

### 🔹 Testing

Реализована полноценная стратегия тестирования:

* **Unit-тесты** для проверки логики компонентов
* **Benchmark-тесты** для измерения производительности
* Оценка пропускной способности системы под нагрузкой

---

## 🎯 Цель проекта

Основная цель — не просто создать планировщик задач, а глубоко понять:

* Как работает конкурентность в Go
* Как эффективно управлять ресурсами
* Как строить отказоустойчивые системы
* Как анализировать и оптимизировать производительность
